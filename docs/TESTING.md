# آلية اختبار صفحات نكفيك

هذه هي المنظومة المعتمدة للتأكد أن جميع صفحات المنصة تعمل، وأن أي تعديل جديد
لا يكسر صفحة قائمة.

## طبقات الاختبار

| الطبقة | ماذا تفحص | متى تعمل |
|---|---|---|
| اختبارات الوحدات (Vitest) | منطق الدوال والمكونات | `npm test` وفي CI |
| اختبارات الصفحات (Playwright) | كل صفحة تفتح في متصفح حقيقي بدون انهيار | `npm run test:pages` وفي CI |
| GitHub Actions | كل ما سبق تلقائياً | عند كل push أو Pull Request |

## اختبارات الصفحات — ماذا تغطي؟

الملف `e2e/pages.spec.ts` يزور **كل مسار** في التطبيق على نسخة الإنتاج
المبنية (نفس ما ينشره Vercel) ويتحقق لكل صفحة من:

1. **التطبيق أقلع**: شاشة "جارٍ تحميل نكفيك" اختفت (أي React اشتغل فعلاً).
2. **لا شاشة خطأ قاتل**: صندوق "تعذّر تشغيل التطبيق" غير ظاهر.
3. **لا صفحة بيضاء**: يوجد محتوى مرئي (أو مؤشر تحميل على الأقل).
4. **لا أخطاء JavaScript** غير معالجة (باستثناء أخطاء الشبكة البيئية).

وتنقسم الصفحات ثلاث فئات في `e2e/routes.ts`:

- `PUBLIC_ROUTES`: الصفحات العامة — يجب أن تعرض محتواها.
- `PUBLIC_PARAM_ROUTES`: صفحات بمعرّف في الرابط — تُزار بقيم غير صالحة
  ويجب أن تعرض حالة "غير موجود" بأدب بدل الانهيار.
- `PROTECTED_ROUTES`: الصفحات المحمية — زيارتها بدون تسجيل دخول يجب أن
  تعيد التوجيه إلى `/login`.

## قاعدة العمل عند إضافة صفحة جديدة

1. أضف المسار في `src/App.tsx` كالمعتاد.
2. **أضف نفس المسار في `e2e/routes.ts`** في الفئة المناسبة.
3. شغّل `npm run test:pages` محلياً قبل الرفع.
4. ادفع التعديل — GitHub Actions سيعيد كل الاختبارات تلقائياً،
   والنتيجة تظهر بعلامة ✓/✗ بجانب الـcommit وفي تبويب Actions.

## الأوامر

```bash
npm test              # اختبارات الوحدات
npm run test:pages    # بناء الإنتاج + اختبار جميع الصفحات في متصفح
npm run test:e2e      # اختبار الصفحات فقط (يتطلب dist مبنياً مسبقاً)
npm run verify        # كل شيء معاً
```

ملاحظة للبيئات المعزولة: إذا كان لديك متصفح Chromium جاهز، مرّر مساره:
`CHROMIUM_EXECUTABLE=/path/to/chromium npm run test:e2e`.

## عند فشل اختبار في CI

- افتح تبويب **Actions** في GitHub → آخر تشغيل أحمر → ستجد اسم الصفحة
  الفاشلة بالعربي في السجل.
- تقرير Playwright الكامل (مع لقطات وأثر التنفيذ) مرفوع كـartifact باسم
  `playwright-report`.

## حدود التغطية الحالية (وخطوات التوسعة القادمة)

الاختبارات الحالية تضمن أن الصفحات **تفتح ولا تنهار**، لكنها لا تختبر بعد:

- تدفقات المستخدم الكاملة (تسجيل → إنشاء فعالية → إصدار تذكرة).
- الصفحات المحمية *بعد* تسجيل الدخول (تحتاج مستخدم اختبار وبيئة Supabase
  منفصلة عن الإنتاج).

هذه هي الخطوة التالية الطبيعية عند الجاهزية.
